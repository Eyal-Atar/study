# Plan 04: Frontend Integration - Startup & Task Completion Wiring

## Objective

Integrate gamification startup sequence into app.js and XP awarding into task completion flow. Wire registerLoginCheckFlow() into initDashboard() to trigger splash/morning prompt. Add fire-and-forget award-xp call in task toggle handler.

Purpose: Connect frontend UI and backend routes. Gamification becomes active on login and task completion without disrupting existing UX.
Output: Integrated startup sequence and task completion wiring.

## Tasks

### Task 1: Wire registerLoginCheckFlow() into app.js initDashboard()

**File:** frontend/js/app.js

1. Import gamification module at top:
   ```javascript
   import { registerLoginCheckFlow } from './profile.js?v=AUTO';
   ```

2. In initDashboard() function, after loadExams() completes, add:
   ```javascript
   // Gamification: splash screen and morning prompt on first login of day
   try {
     registerLoginCheckFlow();
   } catch (e) {
     console.error('registerLoginCheckFlow failed:', e);
   }
   ```

3. Placement: registerLoginCheckFlow() runs fire-and-forget after loadExams(), so calendar is already populated.

**Verify:**
- app.js imports and parses without errors
- registerLoginCheckFlow() called on app startup
- Splash appears on first login when streak >= 3
- Morning prompt appears after splash (if tasks exist)
- Dashboard not blocked by gamification initialization

### Task 2: Wire award-xp call into tasks.js task toggle handler

**File:** frontend/js/tasks.js

1. Find _handleTaskToggle() function (around line 400-500)

2. After PATCH succeeds and block is marked done (isDone = true), add:
   ```javascript
   if (resp.ok && isDone) {
     // Award XP for completed block (fire-and-forget, non-blocking)
     authFetch(`${API}/gamification/award-xp`, {
       method: 'POST',
       headers: { 'Content-Type': 'application/json' },
       body: JSON.stringify({ task_id: taskId, block_id: blockId })
     })
       .then(r => r.json())
       .then(xpResult => {
         if (xpResult && xpResult.xp_earned > 0) {
           console.log('XP awarded:', xpResult);
         }
       })
       .catch(e => {
         console.warn('XP award failed:', e);
       });
   }
   ```

3. Key points:
   - Call inside `if (resp.ok && isDone)` block only
   - Fire-and-forget: no await, .catch() suppresses errors
   - Non-blocking: task completion UX finishes before XP result arrives
   - Log to console for debugging

**Verify:**
- tasks.js imports and parses without errors
- Award-xp POST fires when task block completed
- Request body includes task_id and block_id
- Task completion not blocked if XP call fails (check by making backend unavailable)
- No console errors from XP wiring

## Verification

registerLoginCheckFlow() wired into app.js startup after loadExams(). award-xp call wired into tasks.js toggle handler for completed blocks. Both integrations use fire-and-forget pattern. No existing functionality disrupted.
