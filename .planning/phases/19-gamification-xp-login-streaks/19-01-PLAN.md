# Plan 01: Database Schema & Gamification Utilities

## Objective

Establish the gamification database schema and utility functions. Create three new tables (user_xp, user_streaks, user_badges) and add the xp_awarded column to schedule_blocks. Build utility functions for XP calculation, streak updates, and badge checking.

Purpose: Foundation for all gamification features. XP, streaks, and badges depend on this schema.
Output: Database tables, utility module with calculation logic.

## Tasks

### Task 1: Create gamification database tables and add xp_awarded column

**File:** backend/server/database.py

Add three new tables and one column to schedule_blocks in the init_db() function in backend/server/database.py:

1. Add `xp_awarded INTEGER DEFAULT 0` column to `schedule_blocks` table (after `completed` column) using the existing ALTER TABLE pattern if column doesn't exist via PRAGMA table_info check.

2. Create `user_xp` table with these columns:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id INTEGER NOT NULL UNIQUE
   - total_xp INTEGER DEFAULT 0
   - current_level INTEGER DEFAULT 1 (range 1-50 per 1000 XP per level formula)
   - daily_xp INTEGER DEFAULT 0
   - daily_xp_date TEXT (YYYY-MM-DD in user's local timezone)
   - FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE

3. Create `user_streaks` table with these columns:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id INTEGER NOT NULL UNIQUE
   - current_streak INTEGER DEFAULT 0
   - longest_streak INTEGER DEFAULT 0
   - last_login_date TEXT (YYYY-MM-DD in user's local timezone)
   - streak_broken INTEGER DEFAULT 0 (flag: 1 if broken since last splash)
   - FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE

4. Create `user_badges` table with these columns:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id INTEGER NOT NULL
   - badge_key TEXT NOT NULL (e.g. "iron_will_7", "knowledge_seeker_50")
   - earned_at TEXT DEFAULT (datetime('now'))
   - UNIQUE (user_id, badge_key)
   - FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE

Use the existing migration pattern from Phase 17 with CREATE TABLE IF NOT EXISTS and PRAGMA table_info checks.

**Verify:**
- Run: `python -c "from backend.server.database import init_db; init_db(); print('✓ tables created')"` — succeeds with no errors
- Check schema: `sqlite3 .env.db ".schema user_xp"` — shows all columns and constraints
- Check schedule_blocks: `sqlite3 .env.db "PRAGMA table_info(schedule_blocks);"` — includes xp_awarded column

### Task 2: Create gamification utility module with XP, streak, and badge logic

**Files:** backend/gamification/__init__.py, backend/gamification/utils.py

Create a new backend/gamification/ module with utility functions for XP calculations, streak updates, and badge checking.

1. Create backend/gamification/__init__.py as empty module file.

2. Create backend/gamification/utils.py with these functions:

   a) `calculate_xp(focus_score: int, estimated_hours: float) -> int`:
      - Formula: round(focus_score * estimated_hours * 10)
      - focus_score range: 1-10
      - Return integer XP value

   b) `update_user_xp(db, user_id: int, xp_earned: int, tz_offset: int = 0) -> dict`:
      - Check if user_xp row exists; create if not
      - Get today's date in user's local timezone
      - If daily_xp_date != today, reset daily_xp to 0
      - Add xp_earned to total_xp and daily_xp
      - Recalculate current_level using: level = min(50, floor(total_xp / 1000) + 1)
      - Update user_xp row
      - Return dict with total_xp, current_level, daily_xp, level_up flag

   c) `update_streak(db, user_id: int, tz_offset: int = 0) -> dict`:
      - Get today's date in user's local timezone
      - Check if user_streaks row exists; create if not
      - Determine if first_login_today, streak_broken, and current_streak
      - Return dict with these fields

   d) `check_and_award_badges(db, user_id: int, user_xp_row: dict, streak_row: dict) -> list[str]`:
      - Query existing badge_keys for user
      - Check each badge criterion (streak milestones, task counts, level milestones)
      - INSERT newly earned badges
      - Return list of newly earned badge_keys

**Verify:**
- Run: `python -c "from backend.gamification.utils import calculate_xp; assert calculate_xp(3, 1.0) == 30; print('✓ XP calc')"` — passes
- Import works: all functions callable without errors
- Functions tested with sample data: sample inserts to user_xp, user_streaks work correctly
