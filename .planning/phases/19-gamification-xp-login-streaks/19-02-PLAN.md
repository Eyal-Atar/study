# Plan 02: Gamification Backend Routes & API

## Objective

Create the gamification FastAPI router with four endpoints: login-check (first-of-day gate), award-xp (XP grant + badge unlock), reschedule-task (morning prompt action), and summary (dashboard read). Register the router in the FastAPI app.

Purpose: Backend API interface for all gamification interactions. Handles streak detection, XP grants, badge unlocks, and morning prompt task rescheduling.
Output: Four working API endpoints callable from frontend, integrated into FastAPI app.

## Tasks

### Task 1: Implement POST /gamification/login-check endpoint

Create backend/gamification/routes.py with the login-check endpoint.

- Authentication: Requires valid JWT
- Logic:
  1. Get user's timezone_offset from users table
  2. Call update_streak(db, user_id, tz_offset) from gamification.utils
  3. If first_login_today == False: return early with { first_login_today: False }
  4. If first_login_today == True: get morning prompt tasks, streak info, newly earned badges
  5. Return response with all fields

**Verify:**
- Endpoint callable with valid JWT: returns 200
- Response has expected fields: first_login_today, streak, longest_streak, streak_broken, is_milestone, morning_tasks, badges_newly_earned
- First call of day returns first_login_today=True; second call same day returns False
- Streak increments correctly on consecutive days; breaks and resets after gap > 1 day

### Task 2: Implement POST /gamification/award-xp endpoint

Add award-xp endpoint to routes.py.

- Request body: { task_id: int, block_id: int }
- Logic:
  1. Query schedule_blocks: verify block exists and completed = 1
  2. Check xp_awarded flag: if already set, return { xp_earned: 0 }
  3. Query task: get focus_score, estimated_hours
  4. Calculate XP: calculate_xp(focus_score, estimated_hours)
  5. Update schedule_blocks: set xp_awarded = 1
  6. Call update_user_xp to update total_xp, daily_xp, recalculate level
  7. Call check_and_award_badges to check for new badges
  8. Return response with xp_earned, new_total, new_level, level_up, daily_xp, badges_earned

**Verify:**
- XP calculated correctly: verify xp_earned = focus_score * estimated_hours * 10
- xp_awarded flag set to 1, preventing double award
- Total XP and level updated correctly
- daily_xp increments; level-up detected at 1000 XP boundaries
- Calling twice with same block_id returns xp_earned=0 on second call

### Task 3: Implement POST /gamification/reschedule-task/{task_id} and GET /gamification/summary

Add two more endpoints to routes.py.

**POST /gamification/reschedule-task/{task_id}:**
- Request body: { action: "reschedule" | "delete" | "skip" }
- Logic based on action: reschedule updates day_date=today and regenerates schedule; delete sets status='deferred'; skip makes no change

**GET /gamification/summary:**
- Returns all gamification state: xp data, streak data, all earned badges
- Check daily_xp_date vs today; reset if old
- Return dict with xp, streak, badges objects

**Verify:**
- GET /gamification/summary returns full structure with xp, streak, badges
- POST reschedule-task with action=reschedule rolls over task
- POST with action=delete defers task
- POST with action=skip makes no change
- Capacity check prevents rescheduling when quota exceeded

### Task 4: Register gamification router in FastAPI app

**File:** backend/server/__init__.py

Add import and router registration:
```python
from gamification.routes import router as gamification_router
# ... after other include_router calls:
app.include_router(gamification_router, prefix="/gamification", tags=["gamification"])
```

**Verify:**
- app.py imports without error
- All four endpoints callable via HTTP
- OpenAPI schema includes gamification endpoints

All four gamification endpoints implemented and working. Login-check detects first-of-day, returns streak and morning tasks. Award-xp grants XP and checks badges. Reschedule-task handles morning prompt actions. Summary provides dashboard data. Router registered in FastAPI app.
