# 19.1-01 — Fix XP Circle Display Sync

## Context

The XP circles in the Achievements Tab (daily and overall) are rendered
correctly when the tab is first opened via `initGamification()`, which fetches
`/gamification/summary` and calls `renderAchievementsTab()`. However, when
the user completes a schedule block during the same session, the XP award
fires (via `authFetch /gamification/award-xp`) but the result is only
`console.log`-ged. The exported `updateXPDisplay()` function in `profile.js`
is never called, so the circles stay frozen at their previous values.

### Root Cause (confirmed by code inspection)

File: `frontend/js/tasks.js`, lines 482–497

```js
// Gamification: award XP when a block is marked done (fire-and-forget, non-blocking)
if (patchRes.ok && !isDone && isBlockToggle && blockId) {
    authFetch(`${API}/gamification/award-xp`, { ... })
        .then(r => r.json())
        .then(xpResult => {
            if (xpResult && xpResult.xp_earned > 0) {
                console.log('XP awarded:', xpResult);  // BUG: only logs, never updates UI
            }
        })
        .catch(e => { console.warn('XP award failed:', e); });
}
```

`updateXPDisplay` is exported from `profile.js` but never imported into
`tasks.js`, so the circles are never animated after a block completion.

### Secondary gap

`updateXPDisplay` silently skips the update when `xpResult.xp_earned <= 0`
(idempotent re-toggle). This is correct. However the function also never
updates the `#achievement-streak` element, which stays at 0 after a live
session unless the user closes and reopens the Achievements Tab. Streak is not
expected to change mid-session (it's a login-time value), so this is acceptable
as-is — no additional change needed for streak.

### Task 2 — Call `updateXPDisplay` after successful XP award

**File:** `frontend/js/tasks.js`, lines 482–497

Replace the `console.log('XP awarded:', xpResult)` line with a call to
`updateXPDisplay(xpResult)`. The full block after the edit:

```js
if (patchRes.ok && !isDone && isBlockToggle && blockId) {
    authFetch(`${API}/gamification/award-xp`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ task_id: taskId, block_id: blockId })
    })
        .then(r => r.json())
        .then(xpResult => {
            if (xpResult && xpResult.xp_earned > 0) {
                updateXPDisplay(xpResult);   // FIX: animate circles live
            }
        })
        .catch(e => {
            console.warn('XP award failed:', e);
        });
}
```

`updateXPDisplay` already handles the SVG offset calculation for both daily and
overall circles. It is a no-op if `xp_earned <= 0` (idempotent re-toggle guard
is already inside it). This is a one-line change in the `.then()` handler.

**Expected outcome:** After ticking a block done, both XP SVG circles animate
to the new value within the same session without requiring tab close/reopen.
