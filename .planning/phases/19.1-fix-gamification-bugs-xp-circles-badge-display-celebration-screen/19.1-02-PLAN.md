# 19.1-02 — Fix Badge Grid Rendering and Fetching

## Context

The badge grid (`#achievement-badges`) is empty even though `GET /gamification/summary`
returns the `badges` array and `renderBadgeGrid()` is correctly wired to render it.
The failure mode is a key-name mismatch: the backend awards badges under keys
like `iron_will_7`, `knowledge_seeker_5`, `xp_1000`, but the frontend's
`BADGE_ICONS` and `getBadgeLabel` maps only recognize keys like `streak_7`,
`level_5`, `first_task`. Every badge falls through to the `default` icon (`🎖`)
and no human-readable label.

This is not a fetch bug. `initGamification()` fetches `/gamification/summary`
correctly. `renderBadgeGrid(summary.badges)` receives the array. Each badge
object has shape `{ badge_key: "iron_will_7", earned_at: "2026-03-03" }`. The
badge renders with `getBadgeIcon("iron_will_7")` which returns `🎖` (default)
and `getBadgeLabel("iron_will_7")` which returns `"iron will 7"` (raw
underscore-replace fallback). The grid is not empty — it renders — but with
wrong icons and labels, making it look like nothing meaningful is there (and
unit tests or visual QA would declare it broken).

Additionally the frontend defines badge keys (`first_task`, `ten_tasks`,
`fifty_tasks`) that no backend criterion will ever award in the current
implementation. These are dead display entries until Plan 19.1-03 adds them to
the backend.

### Root Cause

**File:** `frontend/js/profile.js`, lines 10–21 (BADGE_ICONS) and lines 27–40
(getBadgeLabel).

Backend keys (from `backend/gamification/utils.py` `_BADGE_CRITERIA`):
```
iron_will_7, iron_will_30, iron_will_100
knowledge_seeker_5, knowledge_seeker_10, knowledge_seeker_25, knowledge_seeker_50
xp_1000, xp_5000, xp_10000
```

Frontend keys currently defined:
```
first_task, ten_tasks, fifty_tasks, streak_3, streak_7, streak_30, level_5, level_10, level_20
```

Zero overlap. Fix: replace the frontend maps to match the backend keys exactly.
Also add entries for the first-time-achievement keys that Plan 19.1-03 will add
to the backend (`first_task`, `first_login`, `streak_broken_once`, `week_streak`).

### Task 3 — Verify `renderBadgeGrid` renders newly-earned badges from `award-xp`

**File:** `frontend/js/profile.js`, lines 77–95

`renderBadgeGrid` currently only runs when the Achievements Tab opens (inside
`initGamification`). If a badge is earned mid-session from `award-xp`
(returned in `badges_earned`), the grid does not update live.

Add a thin helper `appendBadgesToGrid` that appends newly earned badge cards
without a full re-render, and export it. Wire it into `tasks.js` alongside
`updateXPDisplay`.

Add to `profile.js` after `renderBadgeGrid`:

```js
// ─── Live badge append (called after award-xp returns badges_earned) ──────

export function appendNewBadges(badgeKeys) {
    if (!badgeKeys || badgeKeys.length === 0) return;
    const container = document.getElementById('achievement-badges');
    if (!container) return;

    // Remove empty-state placeholder if present
    const placeholder = container.querySelector('p');
    if (placeholder) placeholder.remove();

    const now = new Date().toISOString().substring(0, 10);
    const newCards = badgeKeys.map(key => `
        <div class="flex items-center gap-3 bg-dark-900/40 rounded-xl p-3 border border-white/5 badge-new-flash">
            <span class="text-2xl">${getBadgeIcon(key)}</span>
            <div class="flex-1 min-w-0">
                <p class="text-sm font-semibold text-white truncate">${getBadgeLabel(key)}</p>
                <p class="text-xs text-white/30">${now}</p>
            </div>
        </div>
    `).join('');

    // Prepend — newest first, matching API order
    container.insertAdjacentHTML('afterbegin', newCards);
}
```

**Expected outcome:** When a badge is earned from block completion, it appears
in the grid immediately if the tab is open, without requiring tab close/reopen.
