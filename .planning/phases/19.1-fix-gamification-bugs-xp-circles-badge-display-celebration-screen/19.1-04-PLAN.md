# 19.1-04 — Add Celebration Screen for Daily Completion

## Context

When the user marks the last incomplete schedule block done for the current day,
they receive no special feedback. The daily progress bar reaches 100% silently.
This plan adds a "Day Complete" celebration modal that fires once per day when
all of today's study blocks are marked done.

The modal is deliberately lightweight: a full-screen flash with confetti (the
`spawnConfetti` function already exists and is called on individual block
completions), a congratulatory message, and an auto-dismiss after 3.5 seconds.
It uses the same modal pattern as `modal-streak-splash`.

### Detection Logic

`updateStats()` in `tasks.js` already calculates `dailyPct` (0–100) by
comparing `todayDoneMin` against `quotaMin`. However, it does not persist
whether the celebration has already been shown today. We need a session-scoped
guard so the celebration fires only once per browser session (not on every
toggle after 100% is reached).

Detection point: inside the block-toggle handler (`_handleTaskToggle`) in
`tasks.js`, after `updateStats()` is called and the PATCH request succeeds,
check whether today's study blocks are all done. Use a module-level boolean
`_celebrationShownToday` as a session guard.

No backend call is needed. The check is purely client-side from in-memory
schedule data (`getCurrentSchedule()`).

### Scope

- Triggered only when a block is marked DONE (not undone)
- Only when all study blocks for today are completed
- Only once per session (guarded by `_celebrationShownToday` flag)
- Fires after a 500ms delay (so the confetti from the individual block finishes
  first)
- Auto-dismisses after 3.5 seconds; also has a manual close button

### Task 2 — Add `showDailyCelebration` function to `profile.js`

**File:** `frontend/js/profile.js`

Add and export a new function at the end of the file, following the same
pattern as `showStreakSplash`:

```js
// ─── showDailyCelebration ─────────────────────────────────────────────────

let _celebrationAutoClose = null;

export function showDailyCelebration() {
    const modal = document.getElementById('modal-daily-celebration');
    if (!modal) return;

    // Show the modal
    modal.classList.add('active');

    // Wire close button
    const closeBtn = modal.querySelector('#btn-close-celebration');
    if (closeBtn) {
        closeBtn.onclick = () => {
            modal.classList.remove('active');
            if (_celebrationAutoClose) clearTimeout(_celebrationAutoClose);
        };
    }

    // Auto-dismiss after 3.5 seconds
    if (_celebrationAutoClose) clearTimeout(_celebrationAutoClose);
    _celebrationAutoClose = setTimeout(() => {
        modal.classList.remove('active');
        _celebrationAutoClose = null;
    }, 3500);
}
```

**Expected outcome:** Calling `showDailyCelebration()` opens the modal, which
auto-dismisses after 3.5s or when the user taps "Keep going".

### Task 3 — Add `checkAndShowDailyCelebration` helper in `tasks.js`

**File:** `frontend/js/tasks.js`

This helper encapsulates the detection logic and the session guard.

1. Add import for `showDailyCelebration` to the existing `profile.js` import
   line (which already imports `updateXPDisplay` and `appendNewBadges` from
   Plans 19.1-01 and 19.1-02):

```js
import { updateXPDisplay, appendNewBadges, showDailyCelebration } from './profile.js?v=AUTO';
```

2. Add a module-level session flag near the top of `tasks.js` (outside any
   function, alongside other module-level state):

```js
let _celebrationShownToday = false;
```

3. Add the helper function (can be placed near `updateStats`):

```js
function checkAndShowDailyCelebration() {
    if (_celebrationShownToday) return;

    const today = getTodayStr();
    const schedule = getCurrentSchedule() || [];

    const todayStudyBlocks = schedule.filter(
        b => b.day_date === today && b.block_type === 'study'
    );

    // Need at least one study block to avoid false-positive on empty days
    if (todayStudyBlocks.length === 0) return;

    const allDone = todayStudyBlocks.every(b => b.completed === 1);
    if (!allDone) return;

    _celebrationShownToday = true;

    // Delay slightly so per-block confetti finishes first
    setTimeout(() => {
        showDailyCelebration();
    }, 500);
}
```

**Expected outcome:** Helper correctly detects when all today's study blocks are
done and fires once per session.

### Task 5 — Reset `_celebrationShownToday` on day change

**File:** `frontend/js/tasks.js`

If the user keeps the app open across midnight, `_celebrationShownToday` should
reset so the next day's celebration can fire. Hook into the existing
day-change detection or add a date check inside `checkAndShowDailyCelebration`:

Replace the flag check at the top of `checkAndShowDailyCelebration` with a
date-aware guard:

```js
// Module-level guard stores both flag and the date it was set for
let _celebrationState = { shown: false, date: null };

function checkAndShowDailyCelebration() {
    const today = getTodayStr();

    // Reset if date has rolled over
    if (_celebrationState.date !== today) {
        _celebrationState = { shown: false, date: today };
    }

    if (_celebrationState.shown) return;

    const schedule = getCurrentSchedule() || [];
    const todayStudyBlocks = schedule.filter(
        b => b.day_date === today && b.block_type === 'study'
    );

    if (todayStudyBlocks.length === 0) return;

    const allDone = todayStudyBlocks.every(b => b.completed === 1);
    if (!allDone) return;

    _celebrationState.shown = true;

    setTimeout(() => {
        showDailyCelebration();
    }, 500);
}
```

Remove the standalone `let _celebrationShownToday = false` from Task 3 since
it is replaced by `_celebrationState`.

**Expected outcome:** Celebration fires at most once per calendar day per
browser session, even if the app is kept open overnight.
